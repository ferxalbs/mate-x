# MaTE X v0.1.2 Release Notes

**Candidate:** `v0.1.2-rc.2`  
**Status:** Internal release candidate — **not** publicly published  
**Qualified platform:** **macOS** (Intel x64 host build; Apple Silicon via Rosetta/universal packaging path deferred if not built on arm64)  
**Deferred:** Windows qualification (architecturally retained, non-blocking)

## What this release is

MaTE X is **not an IDE**. It is the trust layer for AI-written code: a conversational desktop agent that opens a local repository, inspects evidence, applies scoped changes when policy allows, validates deterministically, and **keeps Git commit/push hard-gated** until proof is fresh.

v0.1.2 ships the **Native Engineering System**:

- **`EngineeringTask`** is the sole workflow authority.
- The normal product surface is **conversational**.
- Large persistent EngineeringTask cards are not the primary UI; task details are optional/collapsible.
- **Behavior modes** control autonomy only (not separate workflow engines):
  - **Auto** (default) — scoped autonomy (`auto_scoped`); never unrestricted Full access.
  - **Guided** — inspect freely; contextual approval before mutations.
  - **Review** — read-only.
  - **Custom** — independent toggles for edits, commands, network, Git, and auto-validation.
- Auto may perform safe workspace-scoped edits and validation automatically.
- Git commit and push remain protected in every mode.
- The agent inspects repository evidence before asking questions; questions are reserved for material ambiguity, risk, credentials, destructive behavior, or explicit policy boundaries.
- Internal specifications and plans are not mandatory user ceremony.

## Installation (macOS)

1. Obtain the internal candidate artifacts for `v0.1.2-rc.2` (DMG or ZIP).
2. Verify SHA-256 against `artifacts/rc/v0.1.2-rc.2/SHA256SUMS.txt` (not tracked as source).
3. Open the DMG and copy **MaTE X** to Applications (or use the ZIP extract).
4. Launch MaTE X.
5. Configure a Rainy API v3 key in Settings for cloud-backed agent runs.
6. Open a local repository and work conversationally.

**Signing / notarization:** **not required for v0.1.2** (founder decision). This candidate is an **unsigned** macOS DMG/ZIP. First launch may require a macOS security override (right-click → Open).

## Migration from v0.1.1

- Existing workspaces, conversations, and settings are preserved.
- Legacy Factory-shaped data is decoded **migration-only** into EngineeringTask input.
- Migration is idempotent; malformed legacy records fail safely.
- Legacy readiness/proof is **not** trusted for Git; fresh canonical evidence is required.
- Canonical workspace namespace remains **`.mate-x/`** (not `.matex`).

## Security highlights

- Main-process enforcement for policy, GitGate, and tool authorization.
- Renderer has no direct filesystem/shell/Git authority.
- Auto is workspace-scoped; outside-workspace writes require approval.
- Provider prose cannot authorize evidence transitions or Git unlocks.
- Production bundles exclude tests, fake adapters, QA drivers, and self-test IPC.
