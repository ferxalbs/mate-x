## 4. Shipped test/scaffolding findings (pre-hardening)

1. **CRITICAL — self-test in release binary**  
   Dynamic import of `packaged-self-test` from `main.ts` forced a Vite chunk into ASAR. Env gates (`MATE_X_RELEASE_BUILD`) reduced exploitability but **did not remove attack/maintenance surface**.

2. **CRITICAL — main-process test orchestration**  
   `openGuiAndProbe`, userData isolation, perf sample append, create/recover phase routing lived in production entry.

3. **HIGH — committed machine evidence**  
   Host-specific hashes, timestamps, and failure stderr tails under `artifacts/`.

4. **MEDIUM — FakeAgent in production module**  
   Test-only adapter shipped in the same module as production Rainy adapter.

5. **MEDIUM — ripgrep platform matrix**  
   Config listed all platforms; packageAfterCopy skipped missing packages silently. Now host-constrained + fail-closed on missing host binary.

6. **LOW — colocated qualification tests**  
   Increased cognitive load; moved to `qa/`.
